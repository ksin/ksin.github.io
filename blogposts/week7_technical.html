"Look up issues with SQL (specifically SQL Injection). What is it? Why is it a problem? What can newb programmers do to combat this issue?"

SQL allows you to retrieve and update databse data. Web pages often let users input their own search values to draw and display data from a database using SQL. The fact that SQL statements are text only makes them relatively easy to change dynamically to provide users with selected data. For example: a user input can be stored as a variable which gets added to the SQL statement to retrieve the appropriate data. However, this simplicity is also the cause of some security issues in web applications that use SQL. SQL injection is the term for injecting SQL commands into an SQL statement via web input, that can alter and compromise the security of the database.

Read more on this topic here: http://www.w3schools.com/sql/sql_injection.asp

The main dangers to look out for is Always True statements and Batched SQL statements.

The parameters for protection include "blacklists" of words or characters to search for in SQL input to prevent SQL injection attacks. However, the proven approach to protecting a web site is to use SQL parameters - values that are added to an SQL query at execution time, in a controlled manner. THe key is the @ marker.
